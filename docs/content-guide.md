# SHYBO Content Guide

## 1. Purpose

This document defines how text should be written for the SHYBO construction company website.

The goal is to keep all content:

- clear;
- professional;
- concise;
- credible;
- useful;
- natural in Ukrainian;
- consistent across pages.

The website should not sound like an aggressive advertisement. It should communicate competence, reliability, and transparency.

---

# 2. Language

## 2.1 Primary language

The website content should be written in Ukrainian.

Use modern, natural Ukrainian.

Avoid:

- Russian calques;
- bureaucratic phrases;
- overly formal Soviet-style wording;
- machine-translated phrases;
- unnecessary English words;
- mixed-language UI labels.

## 2.2 Company name

Use:

```text
ШИБО
```

or:

```text
SHYBO
```

Choose one primary visual form and use it consistently.

The Ukrainian name may be used in body text, while the Latin form may remain in the logo if that matches the approved brand asset.

## 2.3 Capitalization

Use sentence case for headings:

```text
Наші послуги
Реалізовані проєкти
Про компанію
```

Avoid unnecessary all caps:

```text
НАШІ ПОСЛУГИ
```

All caps may be used only for very short technical labels or logo text.

---

# 3. Tone of voice

The SHYBO tone should be:

- confident;
- direct;
- calm;
- professional;
- technically competent;
- client-focused.

The company should sound experienced without boasting.

Preferred:

```text
Виконуємо будівельні та ремонтні роботи для приватних і комерційних об’єктів.
```

Avoid:

```text
Ми найкраща будівельна компанія, яка гарантує ідеальний результат кожному клієнту.
```

---

# 4. Core writing principles

## 4.1 Be specific

Preferred:

```text
Виконуємо капітальний і поточний ремонт житлових та комерційних приміщень.
```

Avoid:

```text
Ми виконуємо широкий спектр різноманітних робіт найвищої якості.
```

## 4.2 Keep it concise

A user should understand the point quickly.

Preferred:

```text
Будівництво приватних будинків під ключ.
```

Avoid:

```text
Наша компанія здійснює комплексний процес реалізації будівництва приватних житлових будинків під ключ.
```

## 4.3 Do not exaggerate

Do not use unsupported claims such as:

- найкращі;
- бездоганні;
- №1;
- стовідсоткова якість;
- найнижчі ціни;
- унікальні;
- абсолютна гарантія;
- будь-які роботи будь-якої складності.

Use measurable or verifiable language instead.

## 4.4 Do not invent facts

Never invent:

- years of experience;
- completed project counts;
- client names;
- partner logos;
- awards;
- certificates;
- addresses;
- prices;
- warranties;
- team size;
- equipment;
- reviews;
- working hours;
- phone numbers;
- email addresses.

Use placeholders only during development and mark them clearly.

## 4.5 Focus on the client

Explain:

- what service is provided;
- who it is for;
- what problem it solves;
- what the next step is.

# 6. Headings

## 6.1 Main page heading

The `h1` should explain the value clearly.

Examples:

```text
Будуємо майбутнє. Якісно. Надійно. Вчасно.
```

```text
Будівельні рішення для приватних і комерційних об’єктів
```

```text
Реалізуємо будівельні проєкти від ідеї до завершення
```

Use one strong statement.

## 6.2 Section headings

Preferred:

```text
Наші послуги
Реалізовані проєкти
Нам довіряють
Про компанію
Зв’яжіться з нами
```

Avoid duplicate heading pairs with the same meaning.

Not recommended:

```text
НАМ ДОВІРЯЮТЬ
Ми працюємо для найкращих
```

Use only:

```text
Нам довіряють
```

## 6.3 Heading length

Recommended:

- Hero heading: 4–10 words.
- Section heading: 2–5 words.
- Card heading: 1–4 words.

---

# 7. Paragraphs

## 7.1 Length

Recommended:

- Hero description: 1–2 sentences.
- Section intro: 1–3 sentences.
- Service card: 1 short sentence.
- About section: 2–4 short paragraphs.
- CTA: 1 short supporting sentence.

## 7.2 Structure

Prefer short sentences.

Recommended average:

```text
10–18 words per sentence
```

Avoid paragraphs longer than 4–5 lines on desktop.

---

# 8. Calls to action

## 8.1 CTA principles

A CTA should state a clear action.

Preferred:

```text
Переглянути послуги
Переглянути проєкти
Зв’язатися з нами
Обговорити проєкт
Отримати консультацію
Надіслати заявку
```

Avoid vague wording:

```text
Детальніше
Натисніть тут
Дізнатися більше
Перейти
```

unless context already makes the action completely clear.

## 8.2 Primary and secondary CTA

Hero:

```text
Primary: Наші послуги
Secondary: Зв’язатися з нами
```

Services:

```text
Переглянути всі послуги
```

Projects:

```text
Переглянути всі проєкти
```

Final CTA:

```text
Зв’язатися з нами
```

## 8.3 CTA repetition

Do not repeat the same strong CTA in every section.

One main CTA per section is enough.

# 9. Home page content structure

## 9.1 Hero

Recommended structure:

```text

Будуємо майбутнє. Якісно. Надійно. Вчасно.

Description:
Будівельна компанія ШИБО виконує будівельні та ремонтні роботи для приватних і комерційних об’єктів.

Primary CTA:
Наші послуги

Secondary CTA:
Зв’язатися з нами
```

The exact wording must be reviewed against real company capabilities.

## 9.2 Services

Section heading:

```text
Наші послуги
```

Possible service names:

```text
Будівництво
Комерційне будівництво
Приватні будинки
Ремонтні роботи
Оздоблювальні роботи
Промислові роботи
```

Use only services actually provided by the company.

Service description template:

```text
[Type of work] for [type of object or client].
```

Examples:

```text
Будівництво житлових та комерційних об’єктів.
```

```text
Капітальний і поточний ремонт приміщень.
```

```text
Внутрішні та зовнішні оздоблювальні роботи.
```

## 9.3 Projects

Heading:

```text
Реалізовані проєкти
```

Optional intro:

```text
Приклади об’єктів, над якими працювала команда ШИБО.
```

Only use real projects.

Project card fields:

```text
Назва проєкту
Тип робіт
Місто або область
Рік, if approved
```

Do not invent addresses or client names.

## 9.4 Trusted companies

Heading:

```text
Нам довіряють
```

Do not add another statement with the same meaning.

Only show approved company logos.

## 9.5 Final CTA

Recommended:

```text
Не знайшли потрібну послугу?

Зв’яжіться з нами, і ми підберемо оптимальне рішення для вашого проєкту.

Зв’язатися з нами
```

Alternative:

```text
Потрібна консультація?

Розкажіть про свій проєкт — ми уточнимо завдання та запропонуємо наступний крок.

Обговорити проєкт
```

# 16. SEO writing

## 16.1 Page titles

Format:

```text
[Page topic] | ШИБО
```

Examples:

```text
Будівельна компанія ШИБО
Послуги | ШИБО
Проєкти | ШИБО
Про компанію | ШИБО
Контакти | ШИБО
```

Add the city only after the real service area is confirmed.

## 16.2 Meta descriptions

Recommended length:

```text
120–160 characters
```

Requirements:

- natural language;
- one clear topic;
- real service information;
- no keyword stuffing;
- no invented claims.

Example placeholder:

```text
Будівельна компанія ШИБО: будівельні, ремонтні та оздоблювальні роботи для приватних і комерційних об’єктів.
```

## 16.3 Local SEO

When the real city and service area are confirmed, use them naturally in:

- title;
- description;
- H1 or intro;
- contact details;
- structured data.

Do not repeat the city in every sentence.

## 16.4 Keywords

Keywords should arise naturally from the content.

Avoid:

```text
будівництво Київ ціна недорого найкраща компанія ремонт Київ
```

Prefer:

```text
Виконуємо ремонтні та будівельні роботи для приватних і комерційних об’єктів у [місто/область].
```

# 17. Trust and credibility

## 17.1 Use evidence

Credibility should come from:

- real projects;
- real clients;
- real contact details;
- real process;
- real certificates;
- real reviews;
- clear service descriptions.

## 17.2 Avoid generic trust claims

Avoid:

```text
Нам довіряють тисячі клієнтів.
```

unless verified.

Avoid:

```text
Гарантуємо найвищу якість.
```

Preferred:

```text
Умови, строки та обсяг робіт узгоджуємо до початку реалізації.
```

only if this is actually true.

---

# 18. Reviews

Do not create fake reviews.

Each real review should include, with permission:

- client name or approved identifier;
- project type;
- review text;
- optional date.

Do not rewrite a real review so heavily that it changes the meaning.

---

# 19. Numbers and statistics

Use statistics only when verified.

Examples:

```text
12 років роботи
140 завершених об’єктів
25 фахівців у команді
```

must not appear without confirmed data.

During development use:

```text
[Кількість років буде уточнено]
```

or omit the statistic entirely.

---

# 20. Punctuation and typography

Use Ukrainian typography:

- apostrophe: `’`;
- dash: `—`;
- Ukrainian quotation marks: `« »`;
- non-breaking spaces where useful;
- no space before punctuation;
- one space after punctuation.

Preferred:

```text
Зв’яжіться з нами — ми уточнимо деталі.
```

Avoid:

```text
Зв'яжіться з нами - ми уточнимо деталі .
```

---

# 21. Lists

Use lists for genuinely scannable information.

Preferred list length:

```text
3–6 items
```

Avoid one-item lists or very long lists of similar services.

List items should use parallel grammar.

Example:

```text
- будівництво приватних будинків;
- ремонт комерційних приміщень;
- внутрішні оздоблювальні роботи.
```

---

# 22. Placeholder policy

During development, placeholders must be obvious.

Approved placeholders:

```text
Телефон буде додано
Email буде додано
Адресу буде додано
Опис буде уточнено
Фото проєкту буде додано
```

Do not use believable fake data such as:

```text
+38 (099) 123 45 67
info@shybo.com.ua
м. Київ, вул. Будівельників, 24
```

unless the user confirms it is real.

---

# 23. Content approval checklist

Before publishing any page, confirm:

- Is every factual claim verified?
- Are all services real?
- Are contacts real?
- Are client logos approved?
- Are project images real and authorized?
- Are prices real?
- Are guarantees accurately described?
- Are statistics verified?
- Is the Ukrainian natural?
- Are sentences concise?
- Are headings unique and non-duplicative?
- Is every CTA clear?
- Is there any keyword stuffing?
- Is placeholder text still present?
- Is there fake content that should be removed?
